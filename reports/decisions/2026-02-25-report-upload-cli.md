# 設計決策：report-upload CLI 工具

**日期**: 2026-02-25
**標籤**: #design #tooling #workflow #gdrive #automation

## 問題背景

系統中有 7+ 個腳本重複實現相同的報告上傳邏輯：
- GDrive folder ID 硬編碼在各個腳本中
- 無法自動覆蓋（upsert）已存在的文件
- 每次上傳都要手動指定 folder ID
- 難以維護和擴展

## 解決方案

### 1. 集中配置

**文件**: `~/.openclaw/workspace/config/report-folders.json`

```json
{
  "learning": {
    "localDir": "reports/learning",
    "gdriveFolderId": "1P8Sza6c5DxZ4sWYioYW24CcfZ_R1s9h2"
  },
  "decisions": { ... },
  "observations": { ... },
  ...
}
```

### 2. CLI 工具

**用法**:
```bash
# 上傳到指定類型的 folder
report-upload learning 2026-02-25-oom-analysis.md

# 覆蓋已存在的文件
report-upload decisions report.md --upsert

# 從 stdin 創建並上傳
echo "# Report" | report-upload learning --title "Test" --stdin

# 列出支持的報告類型
report-upload --list
```

### 3. 共享模組

**文件**: `~/.openclaw/workspace/tools/lib/gdrive-upload.js`

提供可被其他腳本 require 的 API：
- `upload(filePath, folderId)` - 上傳
- `upsert(filePath, folderId)` - 刪除已存在 + 上傳
- `findByName(name, folderId)` - 查詢
- `deleteFile(fileId)` - 刪除

## 文件清單

| 文件 | 說明 |
|------|------|
| `config/report-folders.json` | 報告類型 → GDrive folder 映射 |
| `tools/lib/gdrive-upload.js` | 共享上傳模組 |
| `tools/report-upload.js` | CLI 工具 |
| `~/.local/bin/report-upload` | Symlink |

## 優點

1. **單一真相來源** - folder ID 集中管理
2. **Upsert 支持** - 自動處理文件覆蓋
3. **可擴展** - 新報告類型只需更新 config
4. **可重用** - 共享模組可被其他腳本使用
